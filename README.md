# Annotated analytical workflow: maned wolf movement and sarcoptic mange

This repository contains the source files and the minimum input data required to run the numbered analytical workflow for *First insights into the movement patterns of maned wolves with sarcoptic mange*. It includes the three final figure PNGs displayed in the annotated workflow.

## Included data and execution order

The data are stored under short, project-relative paths:

- `data/raw/gps_maned_wolf.rds`: raw GPS data read by `01_data_cleaning.R`.
- `data/processed/gps_no_outliers_final.csv`: cleaned GPS data used directly by `02_home_range_function_ctmm.R` and `04_lulc_prop_TEHS.R`; it is regenerated by script 01.
- `data/raster/mapbiomas_10m_sp_2023-0000000000-0000000000.tif`: required by scripts 04 and 05, but intentionally not versioned on GitHub because of its size. Download the 2023, 10-m LULC raster for São Paulo from [MapBiomas](https://brasil.mapbiomas.org/en/) and save it using this exact path and filename before running those scripts.

Run the scripts in numerical order. Each script writes the derived files required by the following step, so `output/` is created locally as the workflow progresses.

The final figures in `figures/` are static outputs generated previously in the original project. The workflow displays them without rerunning their plotting code.

## Requirements for a full analytical rerun

The complete sequence from scripts 01 through 07 was validated from a clean `output/` directory with R 4.6.1 and JAGS 4.3.1. Install JAGS at the system level, then ensure the following R packages are available: `ctmm`, `lubridate`, `move2`, `tidyverse`, `terra`, `sf`, `jagsUI`, and `rjags`. 

```r
install.packages(c(
  "ctmm", "lubridate", "move2", "tidyverse", "terra", "sf",
  "jagsUI", "rjags"
))
```

Before running scripts 04 and 05, download the MapBiomas raster described above. Run scripts `01_data_cleaning.R` through `07_inspect_TEHS_results.R` in numerical order; the derived `output/` directory is intentionally ignored by Git and will be regenerated locally.
